# Brief — Specification

## Summary

Brief is a markdown convention for shared working memory.

The product is the `.brief/` directory and the rules inside it.
The CLI exists to:
- initialize the workspace
- fetch raw inputs
- report whether the workspace is trustworthy
- migrate older workspaces forward

It should stay small.

## Product Position

Brief is not a planner app.
Brief is not a hidden reasoning engine.
Brief is not a pile of helper commands.

Brief is:
- markdown files agents can read and write directly
- explicit rules in `rules/`
- fetched inputs in `raw/`
- a small CLI that tells the truth about health

## Current Schema

Canonical startup schema:

```text
.brief/
├── priorities.md
├── priorities-raw.md
├── decisions.md
├── decisions-raw.md
├── team.md
├── overrides.md
├── agent-log.md
├── PRIORITIES-HUMAN.md
├── .hash
├── .sources
├── state/
├── people/
├── raw/
└── rules/
    ├── BUILD.md
    ├── EVENING.md
    ├── FETCH.md
    ├── INTERVIEW.md
    ├── MORNING.md
    └── SETUP.md
```

The current startup template generated by `brief init --template startup` is the canonical source of truth for this schema.

## Health States

Before trusting Brief, automation and agents should run:

```bash
brief check --health
```

Valid states:
- `healthy-current-schema`
- `legacy-schema`
- `missing`
- `misconfigured`
- `stale`

### Semantics

#### `healthy-current-schema`
The workspace matches the current schema and can be treated as a current Brief workspace.

#### `legacy-schema`
The workspace looks like an older Brief layout.
It can still be used as context, but not trusted as full modern Brief steering.

#### `missing`
No `.brief/` exists.

#### `misconfigured`
A `.brief/` exists, but required current-schema files or directories are missing.

#### `stale`
The schema is current, but `priorities.md` is older than the relevant raw inputs.

## CLI Contract

Brief intentionally exposes four commands:

```bash
brief init
brief fetch
brief check
brief migrate
```

### `brief init`
Creates `.brief/` using the startup template.

### `brief fetch`
Fetches configured sources into `.brief/raw/`.

Behavior must be health-aware:
- healthy current schema: fetch and point the agent to enrichment flow
- legacy schema: fetch can continue, but must clearly say this is degraded context mode
- missing: fail clearly
- misconfigured: warn clearly
- stale: fetch can continue, but must say enrichment is still needed

### `brief check`
Used for automation and trust inspection.

Supported forms:

```bash
brief check --health
brief check --health --json
brief check --enrichment
brief check
```

#### `brief check --health`
Reports schema trust state.

#### `brief check --enrichment`
Checks whether the priorities artifact is stale relative to fetched raw inputs.

If stale, the guidance should be convention-first:
- read `.brief/rules/BUILD.md`
- read `.brief/raw/`
- update `.brief/priorities.md`

Not command magic.

### `brief migrate`
Upgrades a `legacy-schema` workspace toward the current startup schema.

Supported forms:

```bash
brief migrate --dry-run
brief migrate
```

Requirements:
- dry-run first-class support
- deterministic migration plan
- never silently delete human-authored content
- create missing current-schema structure
- idempotent behavior
- explicit post-migration health output

## Trust Boundary

The trust boundary is simple:

1. Health must be checked before trust.
2. A successful command is not the same thing as a healthy workspace.
3. Legacy workspaces are degraded context mode.
4. The markdown artifacts, not CLI output alone, are the steering layer.

This is the key product rule.
If Brief gets fuzzy about trust, it becomes fake confidence theater.

## Intended Workflow

### Morning / automation loop

```bash
brief check --health
brief fetch
brief check --enrichment
# if stale: agent reads rules + raw and updates priorities.md
```

### Legacy upgrade loop

```bash
brief check --health
brief migrate --dry-run
brief migrate
brief check --health
```

## Rules of the Product

1. **Convention first.** The markdown workspace is the product.
2. **Health before trust.** Agents should know exactly how much trust to place in the workspace.
3. **CLI stays small.** Do not bloat it with convenience commands that duplicate file editing.
4. **Agent does the thinking.** CLI fetches and verifies. The agent reads rules and writes the artifact.
5. **Legacy is explicit.** Do not fake compatibility.
6. **Migration is supported.** Old workspaces should move forward through a real path, not vibes.

## Automation Example

```bash
brief check --health
brief fetch
brief check --enrichment
```

If health is `legacy-schema`, either:
- use Brief as degraded context only, or
- run `brief migrate --dry-run` and `brief migrate`

If enrichment is stale:
- read `.brief/rules/BUILD.md`
- read `.brief/raw/`
- update `.brief/priorities.md`

## Non-Goals

- Do not turn Brief into a task manager.
- Do not hide planning logic inside the CLI.
- Do not let old workspaces pretend they are current.
- Do not add helper commands just because agents can already edit markdown.

## Bottom Line

Brief wins when it is boring and honest.

It should:
- define the convention
- tell the truth about health
- fetch raw state
- help agents trust or distrust the workspace correctly

That version of Brief is useful.
The magical version is not.
