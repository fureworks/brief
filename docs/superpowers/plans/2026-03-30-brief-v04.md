# Brief v0.4 Implementation Plan — Relationships + Agent Integration

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task.

**Goal:** Items are connected. Agents log their actions. Overrides are CLI-driven.

**Architecture:** Relationship graph stored as graph.md with explicit links. Agent log is append-only markdown.

**Tech Stack:** TypeScript, existing modules

---

### Task 1: Relationship graph file

**Files:**
- Create: `src/cli/graph.ts`
- Create: `src/store/graph.ts`
- Modify: `src/index.ts`
- Test: `src/__tests__/graph.test.ts`

- [ ] **Step 1: Write failing test**

Test: `brief graph` with no graph.md returns "No relationships found."
Test: `brief graph add blocks issue:project-c#45 pr:project-c#62` creates a link.
Test: `brief graph query issue:project-c#45` returns related items.

- [ ] **Step 2: Implement graph store**

```typescript
interface GraphLink {
  from: string;     // item ID
  to: string;       // item ID
  type: "blocks" | "caused_by" | "related_to" | "supersedes";
  added: string;    // ISO date
}
```

Store as `graph.md`:
```markdown
# Relationships
- issue:project-c#45 blocks pr:project-c#62 (2026-03-29)
- decision:jwt-auth caused_by meeting:2026-03-25-standup (2026-03-25)
```

- [ ] **Step 3: Implement graph CLI**

```bash
brief graph                                    # show all relationships
brief graph add blocks issue:repo#N pr:repo#M  # add a link
brief graph query issue:repo#N                 # show what this item relates to
brief graph --json                             # machine-readable output
```

- [ ] **Step 4: Run tests, verify passes**
