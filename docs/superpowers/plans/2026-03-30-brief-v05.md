> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task.

**Goal:** Non-CLI users can read the brief. More source adapters available. Field mapping for custom sources.

**Architecture:** Static HTML generated from .brief/ markdown. No server — just `brief serve` that renders and opens browser.

**Tech Stack:** TypeScript, marked (markdown → HTML), node:http (local server)

### Task 2: Source field mapping

**Files:**
- Modify: `src/cli/sync.ts`
- Modify: `src/store/config.ts`
- Test: `src/__tests__/sync.test.ts`

- [ ] **Step 1: Write failing test**

Test: source with `mapping = { title = "name", detail = "description" }` transforms JSON fields correctly.

- [ ] **Step 2: Implement field mapping in sync**

```toml
[[sources]]
name = "linear"
type = "command"
command = "linear issues --json"
target = "priorities"
mapping = { title = "title", detail = "description", priority = "priority_label", status = "state" }
```

The sync engine applies mapping before rendering. Unmapped fields are ignored. This makes any JSON source adaptable without code changes.
